# SSI Backend Deployment Guide

## Quick Start

The backend is configured to deploy automatically to Render.

## Environment Variables

You'll need to set these in the Render dashboard:

| Variable | Description | Example |
|----------|-------------|---------|
| `MONGODB_URI` | MongoDB Atlas connection string | `mongodb+srv://user:pass@cluster.mongodb.net/ssi` |
| `JWT_SECRET` | Secret key for JWT tokens | Auto-generated by Render |
| `NODE_ENV` | Environment mode | `production` |
| `PORT` | Server port | `3000` (auto-set by Render) |

## Deployment to Render

### Option 1: Automatic Deployment (Recommended)

1. Push your code to GitHub
2. Go to [render.com](https://render.com) and sign in
3. Click **New +** ‚Üí **Web Service**
4. Connect your repository: `kash-gg/Project_SSI`
5. Render will auto-detect `render.yaml` and configure everything
6. Add your `MONGODB_URI` in the environment variables
7. Click **Create Web Service**

### Option 2: Manual Setup

1. Go to [render.com](https://render.com)
2. Click **New +** ‚Üí **Web Service**
3. Connect GitHub repository: `kash-gg/Project_SSI`
4. Configure:
   - **Name**: `project-ssi-backend`
   - **Root Directory**: `backend`
   - **Environment**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Plan**: Free
5. Add environment variables (see table above)
6. Deploy!

## After Deployment

1. Copy your Render service URL (e.g., `https://project-ssi-backend.onrender.com`)
2. Update `js/api.js` in the frontend with your actual Render URL
3. Push changes to GitHub
4. Your app on GitHub Pages will now work from anywhere!

## Testing

Visit your Render URL to verify:
- Health check: `https://your-app.onrender.com/health`
- API info: `https://your-app.onrender.com/`
